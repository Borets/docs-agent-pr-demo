# 이 프로젝트에 대하여

이 저장소는 자동화된 문서 에이전트의 풀 리퀘스트 생성을 테스트하고 검증하기 위한 **안전하고 일회용 대상**으로 존재합니다.

## 존재 이유

문서 툴링을 개발하려면 브랜치를 푸시하고, 드래프트 PR을 열고, 차이점을 검사할 수 있는 실제 Git 저장소가 필요합니다. 프로덕션 문서 저장소를 그런 종류의 테스트에 사용하면 PR 큐가 지저분해지고, 의도치 않은 병합이 발생하며, 리뷰어가 피로해질 위험이 있습니다.

이 프로젝트는 문서 에이전트에게 다음을 위한 전용 장소를 제공합니다:

- 올바르게 범위가 지정된 브랜치(`docs-agent/<slug>`)를 생성할 수 있음을 증명합니다.
- 변경 사항이 실제 사용자 대상 문서에 반영되기 전에 드래프트 PR 콘텐츠, 형식, 메타데이터를 검증합니다.
- 엔지니어가 고객에게 보이는 페이지를 건드리지 않고 에이전트 동작을 검토하고 프롬프트를 반복할 수 있도록 합니다.

## 이곳에 있는 것

모든 테스트 콘텐츠는 `content/docs/` 아래에 있습니다. 리뷰어가 문서의 내용보다 에이전트의 결과물에 집중할 수 있도록 파일은 의도적으로 최소화되어 있습니다.

## 이곳에 없는 것

이 저장소는 **어떤 제품 문서의 진실의 출처도 아닙니다**. 여기에 있는 어떤 내용도 공개 문서 사이트에 게시되지 않습니다. 고객 대상 자료에서 이 페이지로 링크하지 마십시오.
