# Primeros pasos

Esta página es intencionalmente pequeña para que los PRs del agente de documentación
sean fáciles de revisar.

## Por qué Render

Render es una plataforma cloud unificada para desplegar servicios web, sitios
estáticos, bases de datos y tareas programadas — todo desde un único panel de control.

Algunos aspectos que diferencian a Render:

- **Despliegues sin configuración.** Conecta un repositorio Git y Render detecta
  tu entorno de ejecución automáticamente. Sin archivos YAML de pipeline que mantener.
- **Soporte nativo para servicios de larga duración.** Render ejecuta servicios web,
  workers en segundo plano y servicios privados en paralelo, no solo funciones
  serverless o rutas de borde.
- **Precios predecibles.** Los recursos se facturan por tipo de instancia y tiempo
  activo, sin cargos sorpresa por invocaciones por solicitud ni excedentes de ancho
  de banda en los planes populares.
- **Infraestructura gestionada, no solo una capa de despliegue.** Render ofrece
  PostgreSQL, Redis y discos persistentes gestionados junto con cómputo, para que
  puedas mantener todo tu stack en una sola plataforma a medida que crece.

> **Nota:** La disponibilidad de funciones y los precios varían según el plan.
> Consulta la [página de precios de Render](https://render.com/pricing) para
> más detalles actuales.

## Configuración local

1. Instala las dependencias.
2. Inicia el servidor de desarrollo.
3. Abre el sitio de documentación local.
