# Render Go

Render 原生支持 Go。你可以将任何 Go 应用部署为 Web 服务、后台 Worker 或 Cron Job，无需额外工具——Render 会自动检测你的 Go 模块、构建二进制文件并运行它。

## 要求

- 仓库根目录（或你指定为根目录的子目录）中存在 `go.mod` 文件。
- 推荐使用 Go 1.20 或更高版本。Render 默认使用最新的稳定版本；如果你的应用有特定需求，可通过 `GOVERSION` 环境变量固定版本。

## 快速开始

1. 将你的 Go 仓库推送到 GitHub 或 GitLab。
2. 在 Render 控制台中，选择 **新建 → Web 服务** 并连接你的仓库。
3. 确认检测到的构建和启动命令（见下文），或根据需要修改。
4. 点击 **创建 Web 服务**。

每次推送到已配置的分支时，Render 会自动构建并部署你的服务。

## 构建与启动命令

Render 会自动检测 Go 项目并设置合理的默认值。

| 设置 | 默认值 |
|---|---|
| **构建命令** | `go build -o server .` |
| **启动命令** | `./server` |

如果 `main` 包位于子目录中，请相应调整构建命令：

```bash
# 适用于 main 包位于 cmd/api 的项目的构建命令
go build -o server ./cmd/api
```

你可以随时在服务的 **设置 → 构建与部署** 中修改这两个命令。

## 固定 Go 版本

在服务上设置 `GOVERSION` 环境变量以指定特定版本：

```
GOVERSION=1.22.4
```

Render 会在构建时安装所请求的版本。该变量必须在下次部署前设置才能生效。

## 环境变量

在 Render 上运行的 Go 应用与在本地一样，通过 `os.Getenv` 读取环境变量。在服务的 **环境** 选项卡中设置变量，或通过 Render 环境组引用它们以跨服务共享值。

你的应用应读取 `PORT` 环境变量以了解要监听的端口：

```go
port := os.Getenv("PORT")
if port == "" {
    port = "8080"
}
http.ListenAndServe(":"+port, nil)
```

Render 会自动为 Web 服务注入 `PORT`。硬编码端口号将导致健康检查失败。

## 健康检查

Render 默认向 `/` 发送 HTTP GET 请求以验证服务是否正常运行。如果你的应用暴露了专用的健康检查端点（例如 `/healthz` 或 `/ping`），请在 **设置 → 健康与告警 → 健康检查路径** 中配置不同的路径。

健康检查在默认超时时间内期望收到 2xx 响应。未能及时绑定到 `PORT` 的服务将被标记为不健康，部署会被回滚。

## 静态文件与资产

如果你的应用提供静态资产，请使用 `//go:embed` 嵌入它们，或将其放置在相对于工作目录的位置。运行时的工作目录是你仓库的根目录。依赖构建机器绝对路径的路径将无法正确解析。

## 私有模块

要从私有仓库获取模块，请添加 `GONOSUMCHECK` 或 `GONOSUMDB` 变量，并通过 `NETRC` 环境变量或通过 secret 文件注入的 `~/.netrc` 文件提供凭据。Render 不会在构建容器之外缓存私有模块的下载，因此每次部署都会重新获取。

## 单仓库（Monorepo）设置

如果你的服务是单仓库中的多个服务之一，请将服务的 **根目录** 设置为包含 `go.mod` 的子目录。Render 将相对于该目录运行所有构建和启动命令。

## Cron Job 与后台 Worker

Go 二进制文件同样适用于 Render 上的后台 Worker 或 Cron Job。创建服务时选择相应的服务类型。Cron Job 不需要绑定到 `PORT`；后台 Worker 仅在同时暴露内部或公开 HTTP 端点时才需要绑定。

## 常见问题

**构建失败，提示 `no Go files in .`**
构建命令在错误的目录中运行。将 **根目录** 设置为包含 `go.mod` 的文件夹，或更新构建命令以指向正确的包路径（例如 `go build -o server ./cmd/myapp`）。

**部署后服务立即重启**
检查你的应用是否从环境中读取 `PORT` 并监听该端口。立即退出的进程（例如因端口已被占用或标志错误）会导致 Render 反复重启它。

**构建时出现模块下载错误**
确认所有依赖项都已列在 `go.sum` 中并已提交。如果你使用私有模块代理，请将 `GOPROXY` 和所需凭据设置为服务的环境变量。
