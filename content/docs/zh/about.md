# 关于本项目

本仓库是一个**安全的、可随时丢弃的目标**，用于测试和验证自动化文档代理的 Pull Request 创建流程。

## 为什么存在

构建文档工具需要一个真实的 Git 仓库来推送分支、开启草稿 PR 并检查差异。将生产文档仓库用于此类测试会带来 PR 队列噪声、意外合并以及审阅者疲劳等风险。

本项目为文档代理提供了一个专属场所，用于：

- 验证能否创建正确命名的分支（`docs-agent/<slug>`）。
- 在任何更改进入真实的面向用户文档之前，核查草稿 PR 的内容、格式和元数据。
- 让工程师在不触碰客户可见页面的情况下审查代理行为并迭代提示词。

## 这里存放什么

所有测试内容位于 `content/docs/` 目录下。文件内容刻意保持最简，以便审阅者专注于代理的输出，而非文档本身的内容。

## 这里不存放什么

本仓库**不是**任何产品文档的权威来源。这里的内容不会发布到任何公开文档站点。请勿从面向客户的材料中链接到这些页面。
